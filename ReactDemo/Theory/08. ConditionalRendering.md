## **Conditional Rendering (Show UI Based on State)**

React UI should change based on **conditions**, not DOM manipulation.

---

## 1Ô∏è‚É£ What is Conditional Rendering?

> Showing or hiding UI **based on state or props**

Example:

* If no notes ‚Üí show message
* If notes exist ‚Üí show list

---

## 2Ô∏è‚É£ The 3 Most Common Patterns

### ‚úÖ 1. Logical AND (`&&`)

```jsx
{notes.length === 0 && <p>No notes yet</p>}
```

Works because:

* `true && JSX` ‚Üí JSX
* `false && JSX` ‚Üí nothing

---

### ‚úÖ 2. Ternary Operator

```jsx
{notes.length === 0 ? (
  <p>No notes yet</p>
) : (
  <NoteList notes={notes} />
)}
```

Use when you have **if/else**.

---

### ‚ùå 3. `if` directly in JSX (NOT allowed)

```jsx
{if (notes.length === 0) {...}}
```

## 3Ô∏è‚É£ Apply It to Our App

### Update `App.jsx`

```jsx
import { useState } from "react";
import Header from "./components/Header";
import NoteInput from "./components/NoteInput";
import NoteList from "./components/NoteList";

function App() {
  const [notes, setNotes] = useState([]);

  function addNote(newNote) {
    setNotes((prevNotes) => [...prevNotes, newNote]);
  }

  return (
    <div>
      <Header />
      <NoteInput onAddNote={addNote} />

      {notes.length === 0 ? (
        <p>No notes yet. Add one!</p>
      ) : (
        <NoteList notes={notes} />
      )}
    </div>
  );
}

export default App;
```

Now:

* Empty app ‚Üí shows message
* Add note ‚Üí list appears automatically

üî• Declarative UI in action

## 5Ô∏è‚É£ Common Conditional Rendering Mistakes

‚ùå Using `notes && ...`

```jsx

```

Why wrong?

* Empty array is truthy
* UI will render incorrectly

Always use:

```jsx
notes.length > 0
```
