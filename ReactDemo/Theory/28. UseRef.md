### `useRef` in React (simple explanation)

`useRef` is a React Hook that lets you **keep a value between renders without causing a re-render**.

Think of it as a **box** where you can store something and React will not reset it on re-render.

---

## Syntax

```js
const myRef = useRef(initialValue);
```

* `myRef.current` â†’ holds the value
* Changing `current` **does NOT re-render** the component

```js
function InputFocus() {
  const inputRef = useRef(null);

  const focusInput = () => {
    inputRef.current.focus();
  };

  return (
    <>
      <input ref={inputRef} />
      <button onClick={focusInput}>Focus</button>
    </>
  );
}
```

## 3ï¸âƒ£ Avoid re-render (difference from `useState`)

### `useState`

```js

setCount(count + 1); // causes re-render
```

### `useRef`

```js
const countRef = useRef(0);
countRef.current += 1; // no re-render
```


## â— Important rules

* Changing `ref.current` âŒ does NOT trigger re-render
* Do NOT read/write refs during render (except `.current`)
* Perfect for **non-UI data**

## useRef vs useState (Interview favorite)

| useRef              | useState        |
| ------------------- | --------------- |
| No re-render        | Re-render       |
| Mutable             | Immutable       |
| Stores DOM / values | Stores UI state |
