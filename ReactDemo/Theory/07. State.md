## 2Ô∏è‚É£ What is State?

### Definition (Simple)

> **State is data that belongs to a component and can change over time**

When state changes:
üëâ React **re-renders** the component.

---

## 3Ô∏è‚É£ `useState` Hook (Basic Syntax)

```jsx
import { useState } from "react";

const [value, setValue] = useState(initialValue);
```

* `value` ‚Üí current state
* `setValue` ‚Üí updates state
* `initialValue` ‚Üí starting value

---

## 4Ô∏è‚É£ Important Rule (MEMORIZE)

‚ùå Never change state directly
‚úÖ Always use setter function

## **Handling Events & Updating State**

This is where **React clicks** for most people.

---

## 1Ô∏è‚É£ Events in React (Basics)

React events look like HTML events, but they are:

* CamelCase
* Passed as functions

### Example

```jsx
<button onClick={handleClick}>Click</button>
```

‚ùå Wrong:

```jsx
<button onClick={handleClick()}>
```

Why?
Because that **calls** the function immediately.

---

## 2Ô∏è‚É£ Controlled Inputs (VERY IMPORTANT)

In React:

> **Input value should come from state**

This is called a **controlled component**.

---

## 3Ô∏è‚É£ Update `NoteInput.jsx`

We will:

* Store input text in state
* Handle typing
* Handle button click

### `NoteInput.jsx`

```jsx
import { useState } from "react";

function NoteInput({ onAddNote }) {
  const [text, setText] = useState("");

  function handleChange(e) {
    setText(e.target.value);
  }

  function handleAdd() {
    if (text.trim() === "") return;

    onAddNote(text);
    setText(""); // clear input
  }

  return (
    <div>
      <input
        value={text}
        onChange={handleChange}
        placeholder="Write a note..."
      />
      <button onClick={handleAdd}>Add</button>
    </div>
  );
}

export default NoteInput;
```

## 4Ô∏è‚É£ Update `App.jsx`

Now `App` will:

* Own notes state
* Pass a function to child

### `App.jsx`

```jsx
import { useState } from "react";
import Header from "./components/Header";
import NoteInput from "./components/NoteInput";
import NoteList from "./components/NoteList";

function App() {
  const [notes, setNotes] = useState([]);

  function addNote(newNote) {
    setNotes((prevNotes) => [...prevNotes, newNote]);
  }

  return (
    <div>
      <Header />
      <NoteInput onAddNote={addNote} />
      <NoteList notes={notes} />
    </div>
  );
}

export default App;
```

üìå Notice:

```js
setNotes(prev => [...prev, newNote])
```

This is the **correct way** to update state.

## 5Ô∏è‚É£ Why This Pattern Matters (CRITICAL)

### Data Flow

```
User types ‚Üí NoteInput (state)
Button click ‚Üí App (state update)
State change ‚Üí UI re-render
```

This is:
üî• **Unidirectional data flow**
