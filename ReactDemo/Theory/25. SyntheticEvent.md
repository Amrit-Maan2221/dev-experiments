# 2ï¸âƒ£ Event Bubbling in React

## What is Event Bubbling?

> When an event happens on a child element, it **travels upward** to parent elements.

### Example (Very Important)

```jsx
function App() {
  return (
    <div onClick={() => console.log("Parent clicked")}>
      <button onClick={() => console.log("Button clicked")}>
        Click Me
      </button>
    </div>
  );
}
```

### Output

```
Button clicked
Parent clicked
```

â¡ï¸ This works **because of event bubbling**

---

## Stop Event Bubbling

```jsx
<button
  onClick={(e) => {
    e.stopPropagation();
    console.log("Only button");
  }}
>
  Click
</button>
```

Now parent will **NOT** trigger.

# 3ï¸âƒ£ Handling Forms in React (MOST IMPORTANT)

In React, forms are usually **controlled components**.

---

## Controlled Input (Standard Way)

```jsx
import { useState } from "react";

function LoginForm() {
  const [email, setEmail] = useState("");

  return (
    <input
      type="email"
      value={email}
      onChange={(e) => setEmail(e.target.value)}
    />
  );
}
```

### Whatâ€™s happening?

* Input value = React state
* React controls the input
* Source of truth = state

---

## Handling Form Submit

```jsx
function handleSubmit(e) {
  e.preventDefault(); // stops page reload
  console.log("Form submitted");
}
```

```jsx
<form onSubmit={handleSubmit}>
  <button type="submit">Submit</button>
</form>
```

# 4ï¸âƒ£ Handling Multiple Inputs (Clean Way)

Instead of multiple states:

```jsx
const [form, setForm] = useState({
  email: "",
  password: ""
});
```

```jsx
function handleChange(e) {
  const { name, value } = e.target;

  setForm((prev) => ({
    ...prev,
    [name]: value
  }));
}
```

```jsx
<input name="email" onChange={handleChange} />
<input name="password" onChange={handleChange} />
```

âœ” This pattern is used in **real projects**

  if (!form.email) newErrors.email = "Email required";
  if (form.password.length < 6)

  setErrors(newErrors);

  return Object.keys(newErrors).length === 0;
}
