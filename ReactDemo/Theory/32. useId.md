# ğŸ”¥ Why useId is useful?

### Problem without `useId`

If you generate IDs randomly, it may create **mismatch in SSR** or duplicate IDs.

Example:

```js
const id = Math.random();  // BAD
```

This will be different on server and client.

---

# âœ… Correct use case

### Example: Label + Input

```js
import { useId } from "react";

function InputField() {
  const id = useId();

  return (
    <div>
      <label htmlFor={id}>Name</label>
      <input id={id} />
    </div>
  );
}
```

This ensures:
âœ” unique ID
âœ” correct accessibility
âœ” no hydration errors
