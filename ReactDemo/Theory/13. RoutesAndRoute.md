# `Routes` & `Route`

We already have:

* `react-router-dom` installed
* `BrowserRouter` wrapped at root

Now we answer ONE question:

> **How do we map URLs to screens (components)?**

---

## 1Ô∏è‚É£ What is `<Routes>`? (Mental Model First)

Think of `<Routes>` as:

> A **decision maker** that looks at the current URL and picks ONE route to render.

It:

* Reads the URL from `BrowserRouter`
* Matches it against defined routes
* Renders the best match

---

## 2Ô∏è‚É£ What is `<Route>`?

A `<Route>` is a **rule**:

```
If URL matches this path ‚Üí render this component
```

Example in English:

> If the URL is `/catalog`, show the Catalog screen.

---

| URL          | Screen    |
| ------------ | --------- |

## 5Ô∏è‚É£ Define Routes in `App.jsx`

Open `src/App.jsx`.

### BEFORE (default)

```jsx
function App() {
  return <h1>Hello World</h1>;
}

export default App;
```

---

### AFTER ‚Äî Add `Routes` & `Route`

```jsx
import { Routes, Route } from "react-router-dom";
import Home from "./pages/Home";
import Catalog from "./pages/Catalog";
import Inventory from "./pages/Inventory";

function App() {
  return (
    <Routes>
      <Route path="/" element={<Home />} />
      <Route path="/catalog" element={<Catalog />} />
      <Route path="/inventory" element={<Inventory />} />
    </Routes>
  );
}

export default App;
```

---

## 6Ô∏è‚É£ What Just Happened (VERY IMPORTANT)

Let‚Äôs slow this down.

### When URL is `/`

* `BrowserRouter` reads URL
* `<Routes>` checks routes
* `/` matches
* `<Home />` renders

### When URL is `/catalog`

* `<Catalog />` renders
* Home is **unmounted**

This is **component mount/unmount**, not page navigation.

## 8Ô∏è‚É£ Key Rules (Memorize These)

* `<Routes>` replaces old `<Switch>`
* Only ONE route renders at a time
* Routes depend on URL
* Routes do NOT fetch data by themselves

---
