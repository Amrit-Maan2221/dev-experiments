## **Props (Component Communication)**

Without props, components are **static**.
Props make components **dynamic and reusable**.

---

## 1Ô∏è‚É£ What are Props?

### Definition (Simple)

> **Props are inputs passed to a component**

Just like function arguments.

---

### Normal JS Function

```js
function greet(name) {
  console.log(name);
}

greet("Amrit");
```

---

### React Component

```jsx
function Greeting(props) {
  return <h1>Hello {props.name}</h1>;
}

<Greeting name="Amrit" />
```

---

## 2Ô∏è‚É£ Props Flow (VERY IMPORTANT RULE)

> üî• **Props flow ONE WAY (Parent ‚Üí Child)**

Child **cannot** modify props.

This makes UI predictable.

---

## 3Ô∏è‚É£ Make `Note` Component Dynamic

### Before

```jsx
<p>This is a note</p>
```

---

### After ‚Äî `Note.jsx`

```jsx
function Note({ text }) {
  return (
    <div>
      <p>{text}</p>
    </div>
  );
}

export default Note;
```

üìå This is **destructuring** props.

---

## 4Ô∏è‚É£ Pass Props from Parent

### `NoteList.jsx`

```jsx
import Note from "./Note";

function NoteList() {
  return (
    <div>
      <Note text="Learn React basics" />
      <Note text="Build Simple Notes App" />
      <Note text="Understand props" />
    </div>
  );
}

export default NoteList;
```

Now each `Note` is **reusable**.

---

## 5Ô∏è‚É£ Props Are Read-Only (IMMUTABLE)

‚ùå Wrong:

```jsx

  text = "Changed";
}
```

This is allowed but not recommonded. React does not want you to do like this.

Props = **input**, not state.
