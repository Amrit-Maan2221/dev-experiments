# 3Ô∏è‚É£ CSS Modules (First Real React Styling)

### What Are CSS Modules?

CSS Modules:

* Scope CSS **to a component**
* Avoid global conflicts
* Compile to unique class names

---

## How CSS Modules Work

File naming is important:

```
Navbar.module.css
```

### Navbar.module.css

```css
.navbar {
  display: flex;
  gap: 16px;
}
```

### Navbar.jsx

```jsx
import styles from "./Navbar.module.css";

function Navbar() {
  return <nav className={styles.navbar}>...</nav>;
}
```

---

## What React Actually Does

```css
.navbar ‚Üí navbar__3xKJ2
```

‚úî No collision
‚úî Scoped
‚úî Predictable

---

## When to Use CSS Modules

‚úÖ Small to medium apps
‚úÖ Clean separation
‚ùå Less flexible for dynamic styles

# 6Ô∏è‚É£ Conditional Styles (Very Important)

### Problem

Active route, error, loading, disabled state.

---

### Conditional class example

```jsx
<button
  className={isLoading ? "opacity-50" : "opacity-100"}
>
  Save
</button>
```

---

### Tailwind + condition (clean pattern)

```jsx
<button
  className={`px-4 py-2 ${
    isActive ? "bg-blue-600" : "bg-gray-400"
  }`}
>
  Submit
</button>
```

This is **daily React work**.

# 7Ô∏è‚É£ Dynamic Classes (Based on Props / State)

```jsx
    status === "active"

    </span>
  );
}
```

Used heavily in:

* Catalog status
* Inventory quantity
* Errors & warnings
